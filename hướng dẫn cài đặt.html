Cách hoạt động

Server C++ chạy ở localhost:8080, cung cấp REST API. Khi mở browser vào http://localhost:8080, server trả về file HTML frontend. Frontend gọi API để hiển thị dữ liệu real-time.

📁 Cấu trúc file

2 file duy nhất cần thiết để chạy toàn bộ ứng dụng

project structure
dtu/
├── server.cpp      ← Backend C++: engine mô phỏng + HTTP server
├── index.html      ← Frontend: dashboard web (Chart.js, thuần HTML/JS)
├── run.sh          ← Script chạy tự động (Linux / macOS)

🐧 Ubuntu / Debian / Linux

Cách nhanh nhất — 1 lệnh

Dùng script tự động. Script sẽ kiểm tra, cài compiler nếu thiếu, compile và mở browser.

bash — Terminal 
# Bước duy nhất: chạy script tự động
chmod +x run.sh && ./run.sh

Hoặc nếu muốn làm từng bước thủ công:

1
Cài g++ (nếu chưa có)
bash
sudo apt update && sudo apt install -y g++ curl
 
# Kiểm tra cài thành công:
g++ --version   # phải hiện: g++ (Ubuntu ...) 11.x.x ...
2
Compile server.cpp
bash
g++ -std=c++17 -O2 -pthread -o dtu_server server.cpp
 
# Nếu không có lỗi → sinh ra file dtu_server (~180KB)
ls -lh dtu_server
3
Chạy server
bash
./dtu_server          # mặc định port 8080
# hoặc đổi port:
./dtu_server 9090     # port 9090
4
Mở ứng dụng trong trình duyệt
bash
xdg-open http://localhost:8080
# Hoặc gõ thủ công vào Chrome/Firefox:
# http://localhost:8080

🪟 Windows 10/11 — Dùng WSL (Windows Subsystem for Linux)
❌

Tại sao không compile native trên Windows?

server.cpp dùng POSIX sockets (sys/socket.h, unistd.h) — là API chuẩn Linux/macOS. MinGW/MSVC trên Windows không có các header này. Giải pháp: dùng WSL — cho phép chạy Ubuntu ngay trong Windows, hiệu năng tốt, miễn phí.

1
Cài WSL2 + Ubuntu (chỉ làm 1 lần, cần restart PC)
Mở PowerShell hoặc Command Prompt với quyền Administrator:
PowerShell (Admin)
wsl --install
# → tự cài WSL2 + Ubuntu 22.04
# → Restart máy tính khi được yêu cầu
💡

Sau khi restart, Ubuntu tự mở và yêu cầu tạo username/password. Nhập tùy ý (username không cần trùng Windows).

2
Mở Ubuntu và cài g++
Tìm "Ubuntu" trong Start Menu, mở terminal Ubuntu. Sau đó:
Ubuntu terminal (WSL)
sudo apt update && sudo apt install -y g++ curl
3
Copy file dự án vào WSL và chạy
Giả sử bạn tải dự án vào thư mục C:\Users\TenBan\Downloads\dtu:
Ubuntu terminal (WSL)
# Di chuyển đến thư mục dự án (đổi TenBan = tên user Windows)
cd /mnt/c/Users/TenBan/Downloads/dtu
 
# Chạy script tự động
chmod +x run.sh && ./run.sh
4
Mở trình duyệt trên Windows
Server chạy trong WSL nhưng Windows có thể truy cập bình thường:
Chrome / Edge / Firefox
# Gõ vào thanh địa chỉ trình duyệt Windows:
http://localhost:8080
✅

Tip: Ghim Ubuntu vào Taskbar

Sau khi cài WSL, tìm "Ubuntu" trong Start → chuột phải → "Pin to taskbar". Lần sau mở nhanh hơn.

💡 Workflow điển hình
step by step
1. Mở trang Dashboard → xem trạng thái khởi đầu (200 sv, kỳ 1, ngày 0)
 
2. Sang trang Điều khiển → nhấn "▶ Chạy 1 ngày" vài lần → quan sát GPA thay đổi
 
3. Nhấn "⏩ Chạy 1 kỳ" → chờ vài giây → Dashboard cập nhật, xem chart GPA trend
 
4. Sang Sinh viên → tìm kiếm "Nguyen" → lọc "Sick" → xem ai đang bị bệnh
 
5. Sang Chính sách → kích hoạt Dịch bệnh 40% → quay Điều khiển → chạy thêm 1 kỳ → so sánh GPA
 
6. Sang Nhập thông tin → thêm sinh viên mình: nhập tên, ngành, GPA 3.8
 
7. Sang Bảng xếp hạng → xem mình có top 10 không 😄
 
8. Sang Thống kê → xem biểu đồ GPA/Stress toàn kỳ, bảng tổng kết

📡 API Reference

Test API bằng curl hoặc bất kỳ HTTP client nào

bash — curl examples
# ── GET endpoints ─────────────────────────────────────────
 
# Trạng thái tổng quan
curl http://localhost:8080/api/status
 
# Danh sách sinh viên (phân trang: page 0, 50 sv/trang)
curl "http://localhost:8080/api/students?page=0&size=50"
 
# Giảng viên
curl http://localhost:8080/api/lecturers
 
# Tòa nhà
curl http://localhost:8080/api/buildings
 
# Sự kiện đã xảy ra
curl http://localhost:8080/api/events
 
# Analytics (GPA trend, thống kê kỳ)
curl http://localhost:8080/api/analytics
 
# Bảng xếp hạng top 20
curl "http://localhost:8080/api/leaderboard?n=20"
 
# Sinh viên bỏ học
curl http://localhost:8080/api/dropouts
 
# Activity log
curl http://localhost:8080/api/log
 
# ── POST commands ─────────────────────────────────────────
 
# Chạy 1 ngày
curl -X POST http://localhost:8080/api/command \
     -H "Content-Type: application/json" \
     -d '{"cmd":"run_day"}'
 
# Chạy 1 học kỳ (90 ngày)
curl -X POST http://localhost:8080/api/command \
     -d '{"cmd":"run_semester"}'
 
# Kích hoạt dịch bệnh (30% sinh viên bị ảnh hưởng)
curl -X POST http://localhost:8080/api/command \
     -d '{"cmd":"epidemic","rate":0.3}'
 
# Cấp học bổng cho SV có GPA ≥ 3.2
curl -X POST http://localhost:8080/api/command \
     -d '{"cmd":"scholarship","minGpa":3.2}'
 
# Đóng/mở căng-tin
curl -X POST http://localhost:8080/api/command -d '{"cmd":"close_cafeteria"}'
curl -X POST http://localhost:8080/api/command -d '{"cmd":"open_cafeteria"}'
 
# Thêm sinh viên mới
curl -X POST http://localhost:8080/api/add_student \
     -H "Content-Type: application/json" \
     -d '{"name":"Nguyen Van A","major":"Computer Science","semester":1,"gpa":3.8}'
 
# Thêm giảng viên mới
curl -X POST http://localhost:8080/api/add_lecturer \
     -d '{"name":"TS. Tran Van B","subject":"AI","experience":10}'
 
# Đổi cấu hình (RESET toàn bộ simulation)
curl -X POST http://localhost:8080/api/config \
     -d '{"numStudents":500,"numSemesters":8,"uniName":"My University"}'

🔧 Xử lý lỗi thường gặp

Các vấn đề phổ biến và cách giải quyết

LỗiNguyên nhânGiải pháp
g++: not found Chưa cài compiler Linux sudo apt install g++
macOS xcode-select --install
sys/socket.h: No such file Đang compile trên Windows native (không dùng WSL) Dùng WSL theo hướng dẫn phần Windows ở trên. Không hỗ trợ MinGW native.
bind failed on port 8080 Port đang bị process khác dùng lsof -i :8080 rồi kill <PID>. Hoặc đổi port: ./dtu_server 9090
cannot execute binary file Chạy binary sai kiến trúc (vd: binary x86 trên ARM) Compile lại từ source: g++ -std=c++17 -O2 -pthread -o dtu_server server.cpp
Frontend hiện trắng / lỗi Server chưa chạy hoặc sai port Kiểm tra server đang chạy: curl http://localhost:8080/api/status
macOS: "dtu_server is damaged" Gatekeeper chặn binary lạ xattr -cr dtu_server rồi chạy lại. Hoặc compile từ source.
Charts không hiện Chưa load được Chart.js (cần internet) Kiểm tra kết nối internet lần đầu. Sau đó thường được cache.
Permission denied: run.sh Chưa cấp quyền thực thi cho script chmod +x run.sh rồi ./run.sh

🏗️ Kiến trúc hệ thống

Class hierarchy và luồng dữ liệu

C++ class hierarchy
// ── Entity (abstract base) ──────────────────────────────────────────────
class Entity
 ├─ Student      // knowledge(0-100), energy(0-100), money, GPA(0-4.0), stress, FSM
 └─ Lecturer     // mood(4 states), rating(1-5), research progress
 
class Building // capacity, occupancy, open/close
 ├─ Classroom   // roomNum, hasAC, hasProjector
 ├─ Cafeteria   // mealsToday, mealPrice, lunchRush
 ├─ Library     // bookCount, studyRooms
 └─ Dormitory   // rooms, fee, wifi
 
// ── Engine & Services ───────────────────────────────────────────────────
class University    // orchestrates 4-tick loop per day
class EventManager  // 14% random events/day, 8 types, 4 scopes
class Statistics    // GPA trends, CSV export, semester aggregates
class HttpServer    // pure POSIX, thread-per-connection, 15 REST endpoints
 
// ── Simulation Loop (per day) ───────────────────────────────────────────
Tick 0 Morning  → attendClass()   // knowledge += lecturer.transfer
Tick 1 Midday   → eatMeal()       // energy +10, money -30K
Tick 2 Afternoon→ work()|social() // 33% làm thêm, 67% học nhóm
Tick 3 Night    → sleep()         // energy +25, stress -10, snapshot
